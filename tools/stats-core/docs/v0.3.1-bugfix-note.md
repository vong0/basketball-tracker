# v0.3.1 bugfix note

Fixed issues found during first user test of the film workflow tracker:

- Add shot now saves correctly. The v0.3 `logActivity(type, id, summary)` parameter accidentally shadowed the `id()` helper, which broke shot/event saving.
- Quick event entry now includes generated quick buttons plus a custom quick-event selector.
- All Players report now shows player names.
- Ratio fields like `2PM/2PA`, `3PM/3PA`, `FTM/FTA`, `Open FGM/FGA`, `Bad Shot FGM/FGA`, and `Transition FGM/FGA` no longer render as `NaN`.
- Standalone app now uses a fresh localStorage key: `bballTrackerData_v031`, so the fixed build opens cleanly instead of reusing broken v0.3 browser state.
